Lab 10: Deployment and Security

Publishing the site, securing selected labs, and preparing auto-deploy

Overview

For Lab 10, I prepared my ITWS site for production deployment on Azure. The work focused on moving from a development setup to a live site with a cleaner homepage flow, selective password protection, HTTPS support, and webhook-based deployment.

What I Added

  • A server root redirect in lab10/index.php so the domain can forward visitors into the site
  • A lab10/README.md file with my FQDN and repository information
  • A webhook template in lab10/deploy.php.example for GitHub auto-deploy
  • A canonical labs/open.php router so every lab now has a stable entry URL
  • Updated project data in Lab 8 so secured labs are marked with lock icons
  • Per-folder .htaccess protection for Labs 1 and 9 instead of protecting the whole site
  • Deployment helper scripts to enable HTTPS and verify the site after pulling on Azure

Security Changes

  • Lab 1 is marked as secure in the projects JSON
  • Lab 9 is marked as secure in the projects JSON
  • The root .htaccess was removed so the entire site is no longer locked
  • Lab-specific .htaccess files are used where authentication is required
  • A dedicated HTTPS setup script can be run on the Azure VM to enable SSL and redirect HTTP traffic
